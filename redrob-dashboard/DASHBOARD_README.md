# 🎯 Redrob AI Recruiter Dashboard

> **India Runs Hackathon — Track 1: Data & AI Challenge**  
> Interactive proof-of-concept demo for the Intelligent Candidate Discovery system.

A live Streamlit dashboard that lets judges (and recruiters) explore the top-100
ranked candidates with full filtering, analytics, and deep-profile views.

---

## ✨ What's in the dashboard

| Tab | What you see |
|-----|-------------|
| **📋 Candidate List** | Ranked cards with scores, badges, reasoning, and CSV export |
| **📊 Analytics** | Score distribution, title breakdown, location map, radar chart, salary scatter |
| **🔎 Deep Profile** | Full profile: skills, career history, education, verified scores, signal breakdown |

### Sidebar filters
- Match score range slider
- Job title multi-select
- Location multi-select
- Notice period tier
- "Open to work only" toggle
- "Verified contact only" toggle

## 🚀 Quick start

### 1. Place files in the same folder

```
your_folder/
├── app.py
├── build_data.py
├── requirements.txt
├── submission.csv        ← your hackathon output
└── candidates.jsonl      ← the dataset (full 100K or subset)
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Build the enriched data file (one-time)

```bash
python build_data.py
# → creates enriched_candidates.csv  (~2 seconds)
```

If your files have different names:
```bash
python build_data.py --submission my_submission.csv --jsonl my_candidates.jsonl
```

### 4. Launch the dashboard

```bash
streamlit run app.py
```

Opens at **http://localhost:8501** automatically.

## 💡 For the hackathon demo

When presenting to judges, walk through these three moments:

1. **Sidebar filter** → set "Open to work only" + "≤30 days notice" → show how
   the list narrows to the most hireable candidates instantly.

2. **Analytics tab → Radar chart** → point out how Top 10 candidates
   consistently outperform Bottom 10 across every behavioral signal,
   not just the match score.

3. **Deep Profile tab** → pick Rank #1, scroll to Career History, show
   how the semantic model caught the right engineer even though their
   job descriptions don't use exact JD keywords.
